# Règles de contribution du projet

Ce document produit par [@n4ssim-dev](https://github.com/n4ssim-dev) (Nassim) et [@mmorkos-cyber](https://github.com/mmorkos-cyber) (Mario) établit les règles de contribution du projet **`collecteur-de-citations`**.

## Convention de nommage : Nom de branche

- `api/nom-de-la-fonctionnalite`
- `html/nom-de-la-fonctionnalite`
- `fix/nom-du-bug`
- `docs/nom-de-la-doc`

## Convention de nommage : Noms des commits

- `<Feat>` Ajout du badge nouvelle version
- `<Fix>` Correction du lien analyse drones
- `<Docs>` Mise à jour du README

## Règles obligatoires pour les pull requests

- Une branche par ticket
- Une pull request par ticket
- Pas de push direct sur `main`/`dev`
